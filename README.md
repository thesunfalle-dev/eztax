# Eztax

Локальный сервис для учета ежемесячных поступлений, пересчета дохода в локальную валюту и расчета налога для Грузии и Беларуси.

## Запуск

```bash
npm run dev
```

Открой:

```text
http://localhost:3000
```

Без переменных окружения сервис использует локальные JSON-файлы в `data/`.
Если заданы `SUPABASE_URL` и `SUPABASE_SECRET_KEY`, данные профилей,
записей и кэш курсов хранятся в Supabase.

Профиль и записи зеркалируются в браузере пользователя, а для локальной
разработки — также в `data/clients/<clientId>/`. Если после паузы Supabase
вернется с пустой историей, Eztax автоматически восстановит записи из
резервной копии этого браузера. Не очищай данные сайта в браузере и используй
тот же браузер/профиль, в котором вносишь налоги.

## Что уже есть

- первый запуск с вводом имени, выбором страны и валюты дохода;
- профиль с редактированием имени, страны и валюты дохода;
- поддержка стран: Грузия и Беларусь;
- поддержка валют дохода: USD и EUR;
- таблица по месяцам: валюта дохода, локальная валюта, налог, курс;
- фильтры по году и месяцу;
- добавление новой записи по сумме дохода и дате поступления;
- редактирование суммы дохода и даты с preview-пересчетом локальной валюты перед сохранением;
- автоматическое определение месяца из даты поступления;
- серверный запрос курса у NBG для Грузии и НБРБ для Беларуси;
- обновление текущего курса при открытии сервиса;
- налог 1% для Грузии и 10% для Беларуси;
- пересчет старых GEL-записей в USD по текущему курсу для грузинского профиля;
- fallback через DNS-over-HTTPS, если обычный DNS не резолвит `nbg.gov.ge`;
- локальный file fallback для разработки;
- продакшен-хранение в Supabase;
- раздельные данные тестеров по браузерному `clientId`;
- экспорт текущей таблицы в CSV;
- импорт истории из CSV, XLSX, XLS или JSON через drag-and-drop в модалке добавления;
- автоматический browser-backup и восстановление истории после пустого ответа Supabase.

## Импорт истории

В модалке добавления поступления есть drag-and-drop поле для истории. Можно
загрузить:

- CSV, экспортированный из Eztax;
- JSON с массивом записей или объектом `{ "entries": [...] }`;
- XLSX/XLS с колонками вроде `date`, `receivedDate`, `month`, `USD`, `EUR`,
  `GEL`, `BYN`, `налог`, `курс`.

Импорт мержит записи по связке `страна + валюта дохода + месяц`: существующий
месяц обновляется, новый добавляется. Если в старом файле есть только сумма в
GEL/BYN без USD/EUR, запись сохраняется как импортированная локальная сумма.

## Supabase

1. Создай Supabase project.
2. Открой SQL Editor и выполни [`supabase/schema.sql`](supabase/schema.sql).
3. В Project Settings → API возьми:
   - Project URL → `SUPABASE_URL`;
   - secret key → `SUPABASE_SECRET_KEY`.
4. Добавь эти переменные локально в `.env` или в Vercel Environment Variables.

`SUPABASE_SECRET_KEY` должен оставаться только на сервере. Не добавляй его
в публичный frontend-код.

## Google sign-in

После включения Google provider в Supabase пользователь может войти с Google
и увидеть одну историю на любом устройстве. Сервер проверяет Supabase session
и использует постоянный Supabase user ID как ключ данных. При первом входе
Eztax переносит историю из резервной копии текущего браузера в аккаунт.

Legacy `SUPABASE_SERVICE_ROLE_KEY` тоже поддерживается, если в проекте еще нет
новых `sb_secret_...` ключей.

## Vercel

Проект готов для Vercel:

- статический frontend лежит в `public/`;
- API routes обрабатываются через `api/[...path].js`;
- серверный обработчик переиспользуется из `server.js`.

Для деплоя укажи в Vercel:

```text
Build Command: пусто
Output Directory: пусто (не указывать public)
Install Command: npm install
Environment Variables:
SUPABASE_URL=...
SUPABASE_SECRET_KEY=...
```

`Root Directory` и `Output Directory` не должны указывать на `public/`:
оставь `Root Directory` корнем репозитория, а `Output Directory` пустым.
Тогда Vercel сам опубликует файлы из `public/` и развернет серверные маршруты
из `api/`.

## Источник курса

Для Грузии сервер запрашивает NBG:

```text
https://nbg.gov.ge/gw/api/ct/monetarypolicy/currencies/en/json/?currencies=USD|EUR&date=YYYY-MM-DD
```

Для Беларуси сервер запрашивает НБРБ:

```text
https://api.nbrb.by/exrates/rates/USD|EUR?parammode=2&ondate=YYYY-MM-DD
```

Если источник курса недоступен, запись не сохраняется, а интерфейс показывает ошибку.

Если проблема только в локальном DNS, сервер автоматически резолвит `nbg.gov.ge`
через DNS-over-HTTPS и все равно запрашивает официальный NBG endpoint.
